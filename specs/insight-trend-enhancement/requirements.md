# 需求文档 - 本周趋势智能增强（Insight Trend Enhancement）

> 版本：v1.1 | 日期：2026-04-07 | 状态：已确认（参考范围数据已验证修正）

## 概述

本次改进针对记录页（`pages/record`）中的**本周趋势模块**（`insight-section` 组件），将现有的「冷数字 + 笼统百分比」展示方式升级为**「语义化状态 + 月龄参考 + 可视化定位 + 一句话智能提示」**模式。核心目标是让新手父母在 3 秒内判断宝宝的喂养、睡眠、排便、体温数据是否在月龄正常范围内，以及当偏离时该如何应对。

**改进价值**：
- ❌ 现状：「日均1次 · 上周1次」→ 新手父母不知道这是好是坏
- ✅ 目标：「正常 ✓ · 日均6次 · 参考6-10次 · 喂养规律，保持即可」→ 一目了然

## 改进范围

| 组件/服务 | 改动类型 | 说明 |
|-----------|----------|------|
| `insight-section` 组件 | **重构** | WXML/WXSS/JS 全面升级卡片内容结构 |
| `trendService.js` | **增强** | 新增月龄参考范围查询 + 智能状态计算 + 提示语生成 |
| `report-popup` 组件 | 不改动 | 保持现有"查看详细报告"功能不变 |

## 用户角色

- **新手父母**：0-12 个月婴儿的初为人父母者，缺乏育儿经验，依赖数据参考判断宝宝是否正常
- **有经验的照护者**：已有一定育儿经验，关注趋势变化而非绝对数值
- **协同照护者**（祖父母等）：偶发查看，需要一目了然的结论性信息

## 功能需求

### FR-1：智能状态标签（替代笼统「需关注」）

**用户故事：** 作为新手父母，我想要看到语义化的状态标签（如"正常""偏少""偏多"），以便直观了解宝宝各项数据是否健康，而不是自己分析一个百分比数字意味着什么。

**验收标准：**
1. When 当前值在月龄参考范围内（FR-2 定义的范围），the system shall 在趋势卡片标题行右侧显示绿色状态标签「正常」
2. When 当前值偏离参考范围 ≤ 30%，the system shall 显示橙色状态标签「偏少」或「偏多」（根据方向判断）
3. When 当前值偏离参考范围 > 30%，the system shall 显示红色状态标签「明显偏少」或「明显偏多」
4. When 体温维度（无参考范围概念），the system shall 根据异常次数显示：0次异常→绿色「正常」；1-2次→橙色「需关注」；≥3次→红色「需就医」
5. While 状态标签显示，the system shall 状态标签替代原有的百分比数字作为视觉焦点（字号 24rpx、圆角胶囊样式），百分比改为次要信息（字号 20rpx、灰色）
6. When 上周和本周均无数据时，the system shall 显示灰色标签「无数据」，不计算状态

**状态标签颜色映射（复用 app.wxss 语义变量）：**

| 状态 | 颜色变量 | 背景变量 |
|------|----------|----------|
| 正常 | `--success-color` #7BC950 | `rgba(123, 201, 80, 0.1)` |
| 偏少/偏多 | `--warning-color` #D4883D | `rgba(212, 136, 61, 0.1)` |
| 明显偏少/明显偏多 | `--danger-color` #E85454 | `rgba(232, 84, 84, 0.1)` |
| 无数据 | `--text-hint` #999999 | `rgba(153, 153, 153, 0.1)` |

### FR-2：月龄参考范围显示

**用户故事：** 作为新手父母，我想要看到宝宝当前月龄对应的正常参考范围，以便有一个"标尺"来衡量日均数据是否合理，而不是靠自己搜索或猜测。

**验收标准：**
1. When 趋势卡片数据加载完成，the system shall 在日均值下方显示参考范围，格式：「参考 X-Y次」或「参考 X-Yh」
2. When 当前宝宝有出生日期，the system shall 根据月龄自动匹配对应的参考范围段（见下方参考数据）
3. When 当前宝宝没有出生日期（数据异常），the system shall 不显示参考范围行，退化为现有展示
4. While 参考范围显示，the system shall 使用 `--text-hint` 色（#999999），字号 22rpx，不抢占日均值的视觉优先级

**月龄参考范围数据（经权威来源验证，详见设计文档 §1）：**

**喂养（日均次数）— 来源：AAP HealthyChildren.org、CDC：**
| 月龄 | 最小值 | 最大值 | 来源依据 |
|------|--------|--------|----------|
| 0-1月 | 8 | 12 | AAP/CDC：新生儿 8-12 次/24h |
| 1-3月 | 6 | 10 | AAP：每 2-3h，约 6-10 次/日 |
| 3-6月 | 5 | 8 | AAP：间隔 3-4h，约 5-8 次/日 |
| 6-12月 | 4 | 6 | AAP：配方奶 4-5 次 + 辅食 |
| 12-24月 | 3 | 5 | AAP：以固体食物为主 |
| 24月+ | 3 | 5 | 常规三餐 + 加餐 |

**睡眠（日均小时数）— 来源：NSF 2015/2025、AASM 2016（AAP 背书）：**
| 月龄 | 最小值 | 最大值 | 来源依据 |
|------|--------|--------|----------|
| 0-1月 | 14 | 17 | NSF 新生儿标准 14-17h |
| 1-3月 | 14 | 17 | NSF 0-3mo 统一标准 |
| 3-6月 | 12 | 16 | NSF 12-15h + AASM 12-16h 取并集 |
| 6-12月 | 12 | 15 | NSF 婴儿标准 12-15h |
| 12-24月 | 11 | 14 | NSF & AASM 一致：11-14h |
| 24月+ | 10 | 13 | NSF 学龄前标准 10-13h |

**排便（日均次数）— 来源：Baaleman 2023 系统综述（J Pediatr）、AAFP POEMs 2024：**
| 月龄 | 最小值 | 最大值 | 来源依据 |
|------|--------|--------|----------|
| 0-1月 | 3 | 8 | 系统综述中位数范围，母乳喂养更频繁 |
| 1-3月 | 2 | 5 | 文献：约 2-3 次/日，范围 1-5 次 |
| 3-6月 | 1 | 4 | 辅食引入前后频率变化 |
| 6-12月 | 1 | 3 | 辅食引入后趋稳 |
| 12月+ | 1 | 3 | AAFP：接近成人模式 |

> **说明**：参考范围经 NSF（美国国家睡眠基金会）、AAP（美国儿科学会）/CDC、AASM（美国睡眠医学会）以及同行评审期刊系统综述验证。详细来源引用见设计文档 `specs/insight-trend-enhancement/design.md` §1。仅用于趋势模块的直观参考指引，不构成医疗建议。

### FR-3：迷你范围定位条（可视化）

**用户故事：** 作为照护者，我想要通过一个简单的可视化进度条看到宝宝日均值在参考范围中的位置，以便不用对比数字就能直觉感知偏离程度。

**验收标准：**
1. When 月龄参考范围数据存在，the system shall 在参考范围文字上方显示一个水平范围条（高度 8rpx，宽度 100%）
2. When 当前日均值在参考范围内，the system shall 范围条中间区段为绿色（`--success-color`），当前值定位点在绿色区段内
3. When 当前日均值低于参考范围，the system shall 范围条左侧高亮橙色/红色，定位点偏左
4. When 当前日均值高于参考范围，the system shall 范围条右侧高亮橙色/红色，定位点偏右
5. While 范围条显示，the system shall 使用圆角（4rpx）样式，背景为 `--bg-primary`（#F5F1EB），与卡片融合
6. When 无月龄参考范围数据（出生日期缺失），the system shall 不显示范围条
7. While 定位点显示，the system shall 为实心圆点（宽 12rpx、高 12rpx），颜色与状态标签一致（正常=绿色、偏离=橙色、严重偏离=红色）

**范围条布局示意：**
```
[   ●───────■■■■■■■■■───]   正常（点在中间绿色区域）
[●──■■■───────────────────]   偏少（点在左侧橙色区域）  
[───────────────────■■■──●]   偏多（点在右侧橙色区域）
```

### FR-4：一句话智能提示

**用户故事：** 作为新手父母，我想要在每个趋势卡片底部看到一句简短的指导性建议，以便知道当前数据意味着什么以及是否需要采取行动，而不只是看到冰冷的数字。

**验收标准：**
1. When 趋势卡片数据正常（状态标签为「正常」），the system shall 在卡片底部显示正面肯定的提示语（如"喂养规律，保持即可"）
2. When 趋势卡片数据偏离（状态标签为「偏少/偏多」），the system shall 显示温和的关注提示（如"日均睡眠略低，建议关注夜间作息"）
3. When 趋势卡片数据严重偏离（状态标签为「明显偏少/明显偏多」），the system shall 显示建议行动提示（如"排便次数明显减少，如持续请咨询医生"）
4. When 趋势卡片无数据，the system shall 显示引导提示（如"开始记录喂养数据，获取趋势分析"）
5. While 提示语显示，the system shall 字号 22rpx，颜色为 `--text-secondary`（#666666），单行显示（超长截断）
6. While 提示语生成，the system shall 基于本地规则引擎自动生成（不调用 AI），确保即时响应

**提示语规则引擎（各维度）：**

**喂养提示：**
| 条件 | 提示语 |
|------|--------|
| 正常 | 「喂养规律，保持即可 👍」 |
| 偏少 | 「日均喂养略少，注意宝宝饥饿信号」 |
| 偏多 | 「喂养频率偏高，可观察是否吃饱」 |
| 明显偏少 | 「喂养次数明显偏少，建议关注」 |
| 明显偏多 | 「频繁喂养，建议咨询是否需调整」 |
| 无数据 | 「开始记录喂养，获取趋势分析」 |

**睡眠提示：**
| 条件 | 提示语 |
|------|--------|
| 正常 | 「睡眠充足，继续保持作息规律」 |
| 偏少 | 「日均睡眠略低，关注夜间作息」 |
| 偏多 | 「睡眠偏多，注意观察精神状态」 |
| 明显偏少 | 「睡眠明显不足，建议改善睡眠环境」 |
| 明显偏多 | 「嗜睡需关注，如持续请咨询医生」 |
| 无数据 | 「开始记录睡眠，了解作息规律」 |

**排便提示：**
| 条件 | 提示语 |
|------|--------|
| 正常 | 「排便正常，消化良好」 |
| 偏少 | 「排便次数略少，多注意饮食」 |
| 偏多 | 「排便次数偏多，注意大便性状」 |
| 明显偏少 | 「排便明显减少，如持续请咨询医生」 |
| 明显偏多 | 「腹泻风险，注意补水和就医」 |
| 无数据 | 「开始记录排便，跟踪消化情况」 |

**体温提示：**
| 条件 | 提示语 |
|------|--------|
| 正常（0次异常） | 「体温正常，宝宝很健康」 |
| 需关注（1-2次异常） | 「有体温偏高记录，注意观察」 |
| 需就医（≥3次异常） | 「多次发热，建议及时就医」 |
| 无数据 | 「定期测量体温，关注健康状况」 |

### FR-5：趋势卡片信息架构重组

**用户故事：** 作为照护者，我想要每张趋势卡片以"结论先行、数据支撑、建议收尾"的信息层次展示，以便快速获取最重要的信息。

**验收标准：**
1. When 趋势卡片渲染，the system shall 按以下垂直信息层次排列（自上而下）：
   - **第一行**：图标 + 类型名称 + 状态标签（FR-1）
   - **第二行**：迷你范围定位条（FR-3）
   - **第三行**：日均值 + 参考范围（FR-2） + 环比变化（百分比，次要）
   - **第四行**：一句话智能提示（FR-4）
2. When 卡片布局渲染完成，the system shall 保持 2×2 网格布局不变（`grid-template-columns: 1fr 1fr`）
3. While 卡片渲染，the system shall 维持与现有 `insight-section` 一致的外部尺寸和间距，不影响记录页整体布局
4. When 趋势数据正在加载（`loading === true`），the system shall 每个卡片内显示骨架屏占位（4 行条状 shimmer）

**单个卡片布局结构：**
```
┌───────────────────────────┐
│ 🍼 喂养          正常 ✓  │  ← 第一行：图标+名称+状态标签
│ [━━━━━●━━━━━━━━━━]       │  ← 第二行：迷你范围条
│ 日均6次  参考6-10  ↑8%   │  ← 第三行：日均+参考+环比
│ 喂养规律，保持即可 👍     │  ← 第四行：智能提示
└───────────────────────────┘
```

### FR-6：周环比变化展示优化

**用户故事：** 作为照护者，我想要周环比变化以更温和的方式展示，以便不因大百分比数字产生不必要的焦虑。

**验收标准：**
1. When 周环比变化百分比 ≤ 10%，the system shall 显示为灰色文字 `--text-hint`，表示变化不大（如 "↑5%"）
2. When 周环比变化百分比 11%-30%，the system shall 显示为绿色或橙色（上升绿、下降橙），字号 20rpx
3. When 周环比变化百分比 > 30%，the system shall 显示为对应状态色（正常范围内仍为绿色/橙色；超出范围则为红色），字号 20rpx
4. While 环比数据显示，the system shall 作为第三行最右侧的辅助信息，不再作为卡片的视觉焦点
5. When 上周数据为 0（无法计算环比），the system shall 显示 "新" 标签代替百分比（表示本周首次有数据）

## 非功能需求

### NFR-1：性能要求
- 月龄参考范围查询为纯本地计算（JS 对象查表），耗时 < 1ms
- 智能提示语生成为纯本地规则引擎，不调用 AI 接口，耗时 < 1ms
- 范围条定位计算为纯数学运算，不触发额外 DOM 重排
- 趋势数据加载整体耗时不因本次改动增加（保持现有 30s 缓存策略）

### NFR-2：兼容性要求
- 兼容现有美拉德色彩设计系统（所有新增颜色复用 `app.wxss` 已有变量）
- 兼容微信小程序基础库 ≥ 2.20.0
- 2×2 网格布局在 320px-414px 屏幕宽度范围内不溢出、不换行
- 范围条在低端机型上可降级为纯色条（无渐变），保证性能

### NFR-3：数据一致性
- `trendService.js` 现有的周环比计算逻辑保持不变，新增逻辑为增量扩展
- 月龄参考范围数据内置在 `trendService.js` 中，不依赖外部配置文件（避免引入新的依赖）
- 状态标签的判断逻辑基于日均值 vs 参考范围的比值，与现有的 20% 变化阈值解耦（两者独立判断）

### NFR-4：可维护性
- 月龄参考范围数据以配置对象形式组织，后续可方便调整阈值
- 提示语规则以映射表形式组织，后续可方便扩展和修改文案
- 状态标签的偏离比例阈值（30%）以常量定义，不硬编码在判断逻辑中

### NFR-5：用户体验
- 所有新增视觉元素使用 `app.wxss` 中已有的 CSS 过渡变量（`--transition-fast` 0.2s）
- 状态标签胶囊使用圆角 `--radius-sm`（12rpx），与设计系统一致
- 提示语不使用 emoji 作为图标（遵守 UI 设计规范），但允许在文末使用简单表情符号（如 👍）作为情感点缀
- 卡片内各行之间间距为 `--spacing-xs`（8rpx），保证信息密度适中不拥挤

## 边界条件和异常处理

| 场景 | 处理方式 |
|------|----------|
| 宝宝无出生日期（`birthDate` 为空） | 不显示参考范围、不显示范围条、状态标签退化为现有的百分比变化模式（≥20% 显示"需关注"） |
| 本周和上周均无某类记录 | 显示灰色「无数据」标签 + 引导提示语，范围条不显示，日均值显示为 0 |
| 月龄超过 36 个月（PRD 目标用户上限） | 使用最后一档参考范围（24月+），不报错 |
| 日均值恰好等于参考范围边界值 | 判定为「正常」（含边界） |
| 参考范围上下限相同（极端情况） | 范围条整段显示为绿色，定位点居中 |
| `trendService.js` 缓存数据与最新月龄不一致（跨月场景） | 缓存 30s 过期后自动刷新；月龄计算使用实时 `Date.now()` 而非缓存时间 |
| 体温维度无参考范围概念 | 体温卡片不显示参考范围行和范围条，仅保留状态标签和智能提示 |

## 与现有模块的关系

| 现有模块 | 关系 | 说明 |
|----------|------|------|
| `insight-section` 组件 | **本次改动主体** | WXML/WXSS/JS 全面重构 |

| `app.wxss` 设计系统 | **复用** | 所有颜色/间距/圆角/阴影变量均复用现有定义，不新增 CSS 变量 |
