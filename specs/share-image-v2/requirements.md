# 需求文档 - 成长报告分享图 V2（成绩单式）

## 概述

对现有成长报告分享图进行全面升级，从当前"体检报告式"纯数字堆砌，升级为**「宝宝成绩单」**风格——以方案 A（成绩单式）为基础，融入方案 B（温情日记式）的叙事温度和方案 C（数据可视化式）的视觉冲击力。

**核心价值：**
- 让不懂育儿数据的家人（爷爷奶奶、朋友）也能一目了然地看懂宝宝状态
- 利用项目中已有但分享图未使用的丰富数据（趋势、参考范围、里程碑、疫苗、生长曲线）
- 增强情感温度和社交分享动力

## 用户角色

- **记录者（爸爸/妈妈）**：生成并分享宝宝的周/月成绩单，希望展示完整的成长状态
- **接收者（爷爷奶奶/亲友）**：收到分享图后能直观理解宝宝状况，无需育儿专业知识

## 功能需求

---

### FR-1: 标题区温情化

**用户故事：** 作为记录者，我希望分享图标题有温度感，而不是冷冰冰的"宝宝成长报告"，以便让接收者感受到关爱。

**验收标准：**
1. When 生成分享图时, the system shall 将标题从"宝宝成长报告"改为"**{宝宝名}的一周成绩单**"（周报）或"**{宝宝名}的月度成绩单**"（月报）
2. When 显示日期范围时, the system shall 同时显示"**出生第 N 天**"的天数标记（从 `babyInfo.birthDate` 计算）
3. When 宝宝名字超过 4 个字时, the system shall 截断为前 4 个字 + "…"，确保标题不溢出

---

### FR-2: 综合评分可视化

**用户故事：** 作为接收者，我希望综合评分有直观的可视化表达（而非孤零零的数字），以便快速感知宝宝状态好坏。

**验收标准：**
1. When 显示综合评分时, the system shall 在分数下方绘制一个**弧形进度条**（或线性进度条），用颜色表示等级：
   - 90-100：绿色，显示"非常棒！"
   - 80-89：蓝绿色，显示"状态良好"
   - 70-79：橙色，显示"部分可优化"
   - 60-69：橙红色，显示"需要关注"
   - <60：红色，显示"建议咨询医生"
2. When 绘制进度条时, the system shall 进度条填充比例 = 分数 / 100

---

### FR-3: 四维指标卡 — 状态标签 + 范围条 + 智能提示

**用户故事：** 作为接收者，我希望每个指标（喂养/睡眠/排便/体温）都有清晰的状态评价，而不仅仅是数字，以便判断宝宝是否正常。

**数据源：** 复用 `TrendService` 的 `calculateStatus()`、`getReferenceRange()`、`calculateRangeBarPosition()`、`generateTip()`、`getStatusDisplay()` 方法

**验收标准：**

#### 3.1 状态标签
1. When 绘制喂养/睡眠/排便指标卡时, the system shall 在指标标题旁绘制**状态标签色块**，显示 TrendService 计算出的状态文案：
   - 正常 → 绿色背景 + 白字"正常"
   - 偏少/偏多 → 橙色背景 + 白字"偏少"/"偏多"
   - 明显偏少/明显偏多 → 红色背景 + 白字"明显偏少"/"明显偏多"
2. When 绘制体温指标卡时, the system shall 显示体温专属状态：正常/需关注/需就医
3. If 无数据（noData状态）, then the system shall 显示灰色"无数据"标签

#### 3.2 月龄参考范围条
4. When 宝宝有出生日期且能计算月龄时, the system shall 在喂养/睡眠/排便卡片中绘制一个**迷你范围条**（range bar）：
   - 范围条总宽度约 300px，分为三段：偏低区（灰色）+ 正常区（绿色/浅绿）+ 偏高区（灰色）
   - 在范围条上绘制一个**圆形定位点**，位置由 `calculateRangeBarPosition()` 返回的 `position` 值（0-100）决定
   - 定位点颜色与状态标签一致（正常=绿色，偏低=橙色，偏高=橙色，明显偏=红色）
5. When 绘制范围条时, the system shall 在条的右侧或下方显示参考范围文字，如"参考 4-6 次"（喂养）、"参考 12-15h"（睡眠）

#### 3.3 智能提示语
6. When 绘制每个指标卡时, the system shall 在卡片底部显示一句话智能提示（来自 `TrendService.generateTip()`），如"喂养规律，保持即可 👍"
7. When 提示语包含 emoji 时, the system shall 保留 emoji 显示（Canvas 支持的情况下），或降级为纯文字

#### 3.4 周环比变化
8. When 有上周数据时, the system shall 在日均值旁显示周环比变化，如"↑12%"（上升）或"↓8%"（下降），并用颜色区分方向

---

### FR-4: 生长数据模块（身高/体重）

**用户故事：** 作为记录者，我希望分享图中展示宝宝最近一次的身高体重数据和 WHO 百分位评价，以便让家人了解宝宝的生长发育水平。

**数据源：** 查询 `recordType: 'growth'` 的最近一条记录 + `config/who-standards.js` 的 P3/P15/P50/P85/P97 百分位数据

**验收标准：**
1. When 宝宝有生长数据记录时, the system shall 在分享图中增加一个"生长发育"模块，展示：
   - **体重**：最近一次值 + WHO 百分位评价（如"7.5kg · P50 达标"）
   - **身高**：最近一次值 + WHO 百分位评价（如"68cm · P85 偏高"）
2. When 计算百分位评价时, the system shall 根据宝宝性别（`baby.gender`）和月龄选择对应的 WHO 标准表，判断当前值落在哪个百分位区间：
   - < P3 → "偏低"（红色）
   - P3 - P15 → "偏低正常"（橙色）
   - P15 - P85 → "达标"（绿色）
   - P85 - P97 → "偏高正常"（蓝色）
   - > P97 → "偏高"（红色）
3. When 显示百分位时, the system shall 用简明的文字 + 色块呈现，如 `[绿色块] P50 达标`
4. If 宝宝没有任何生长数据记录, then the system shall 不显示此模块（跳过），不留空白区域
5. When 生长数据记录超过 30 天, the system shall 在数据旁标注"（X天前测量）"提示数据时效性

---

### FR-5: 疫苗接种进度模块

**用户故事：** 作为记录者，我希望分享图中展示宝宝的疫苗接种进度，以便让家人了解疫苗是否按时接种。

**数据源：** 复用 `TodoService` 的 `getTodoStats()` 返回的 `vaccineItems`，或直接查询 `vaccine_records` 集合

**验收标准：**
1. When 宝宝有疫苗计划时, the system shall 在分享图中增加一个"疫苗进度"小模块，显示：
   - 已接种数 / 应接种总数（基于当前月龄）
   - 进度条可视化（如 `████████░░ 8/12`）
   - 若有逾期未接种，用红色标注"X 剂逾期"
2. When 所有应接种疫苗均已完成, the system shall 显示"✅ 疫苗接种进度：全部完成"
3. If 宝宝月龄 < 1 个月（尚无应接种疫苗）, then the system shall 不显示此模块

---

### FR-6: 里程碑达成模块

**用户故事：** 作为记录者，我希望分享图中展示宝宝最近达成的发育里程碑，以便让家人分享成长的喜悦。

**数据源：** 查询 `milestone_records` 集合获取已达成的里程碑，对比 `TodoService._getMilestoneDefinitions()` 获取定义

**验收标准：**
1. When 宝宝有已达成的里程碑记录时, the system shall 在分享图中展示"成长里程碑"模块，显示**最近 3 个**已达成的里程碑名称，如：
   - ✅ 独坐 · ✅ 翻身 · ✅ 社会性微笑
2. When 有待达成（已到窗口期但未完成）的里程碑时, the system shall 显示下一个待解锁的里程碑名称（灰色/虚线），如：
   - 🎯 下一个：爬行
3. If 宝宝没有任何里程碑记录, then the system shall 不显示此模块

---

### FR-7: 本周成就模块

**用户故事：** 作为记录者，我希望分享图中有"成就徽章"增强成就感和持续记录的动力，以便激励自己和家人。

**验收标准：**
1. When 生成分享图时, the system shall 根据本周数据计算并展示最多 3 个成就亮点，可能包括：
   - "📝 连续 N 天坚持记录"（本周期内有记录的天数 ≥ 5 天）
   - "🍼 喂养规律达标"（日均喂养次数在参考范围内）
   - "😴 睡眠充足"（日均睡眠时长在参考范围内）
   - "🌡️ 体温全部正常"（无异常体温记录）
   - "📊 排便规律"（日均排便在参考范围内）
   - "💉 疫苗按时接种"（无逾期疫苗）
2. When 没有任何成就可展示（如全部为"偏少/偏多"）, the system shall 改为显示一句鼓励语，如"每一天的记录都是对宝宝的爱 ❤️"
3. When 成就计算完成时, the system shall 每个成就用一行文字+前缀 emoji 展示，整体视觉轻量

---

### FR-8: AI 建议精简化

**用户故事：** 作为接收者，我不希望看到大段文字的 AI 建议，希望精简为 1-2 句关键建议。

**验收标准：**
1. When 生成分享图中的 AI 建议时, the system shall 将建议文本精简为**不超过 2 句话**（约 60 字以内），取原始 AI 评语中**最重要的一条建议**
2. When AI 评语有多段时, the system shall 按优先级选择：先取"需要关注"类（偏少/偏多状态的维度），再取正面肯定类
3. When 精简后的文本绘制时, the system shall 保留"AI 育儿建议"标题，但卡片高度大幅缩小

---

### FR-9: 每日记录密度条（7日热力简图）

**用户故事：** 作为接收者，我希望看到宝宝这一周每天是否都有被照顾记录，以便了解记录的连续性。（方案 C 亮点元素）

**验收标准：**
1. When 生成周报分享图时, the system shall 在统计卡片下方绘制一行 7 个小方块，代表周一到周日：
   - 有记录的天：填充色块（美拉德色深浅，记录越多颜色越深）
   - 无记录的天：灰色空心方块
2. When 显示日期标签时, the system shall 在方块下方标注简写（一/二/三/四/五/六/日）
3. When 是月报时, the system shall 不显示此模块（月报数据量太大，热力图不直观）

---

### FR-10: 底部品牌区优化

**用户故事：** 作为产品方，我希望底部品牌区更有辨识度和引导性。

**验收标准：**
1. When 绘制底部区域时, the system shall 保留"Baby Care Tracker"品牌名
2. When 绘制底部区域时, the system shall 将副标题从"扫码记录宝宝成长点滴"改为更具行动力的"记录每一天的成长"

---

### FR-11: 分享图整体布局与动态高度

**用户故事：** 作为技术实现者，我需要确保新增模块后分享图的整体布局合理、高度自适应。

**验收标准：**
1. When 所有模块都有数据时, the system shall 按以下顺序从上到下排列：
   ```
   ① 标题区（名字 + 日期 + 出生第N天）
   ② 宝宝信息卡（头像 + 名字 + 月龄 + 综合评分进度条）
   ③ 四维指标卡（喂养/睡眠/排便/体温，每个含状态标签+范围条+提示语）
   ④ 每日记录密度条（仅周报）
   ⑤ 生长发育模块（身高/体重 + WHO百分位）
   ⑥ 疫苗接种进度
   ⑦ 里程碑达成
   ⑧ 本周成就
   ⑨ AI 建议（精简版）
   ⑩ 底部品牌区
   ```
2. When 某模块无数据时, the system shall 跳过该模块，不留空白，后续模块上移
3. When 计算 Canvas 总高度时, the system shall 在绘制前遍历所有模块，累加各模块高度，确保底部 Footer 在正确位置
4. When 分享图总高度超过 2000px 时, the system shall 确保 Canvas 内存不超限（DPR 仍限制为 2）

## 非功能需求

### NFR-1: 性能
- Canvas 绘制总耗时 ≤ 3 秒（含数据查询）
- 导出图片 ≤ 800KB（JPG quality 0.85，DPR 2）
- 新增的数据库查询（growth records、vaccine records、milestone records）应并行执行

### NFR-2: 兼容性
- 支持微信小程序 Canvas 2D API
- 在 iOS / Android 设备上 emoji 可正常渲染（降级方案：emoji 替换为 Canvas 可绘制的图标）
- DPR 1x/2x/3x 设备表现一致（3x 设备限制为 2x 绘制）

### NFR-3: 可维护性
- 新增模块作为独立绘制方法（如 `_drawGrowthSection()`、`_drawVaccineProgress()`），便于后续增删
- 颜色/字号/间距统一使用 `CANVAS_CONFIG` 常量

### NFR-4: 数据安全
- 分享图中不展示宝宝精确出生日期（仅显示月龄和出生天数）
- 不展示家庭地址、家长手机号等敏感信息
- 不展示疫苗的具体品牌或批号

## 边界条件和异常处理

| 场景 | 处理方式 |
|------|---------|
| 宝宝无出生日期 | 不显示月龄参考范围条、百分位评价；状态标签也跳过（因无法计算参考范围） |
| 本周 0 条记录 | 显示"本周暂无记录"提示，四维指标卡显示"--"，跳过成就模块 |
| 从未记录生长数据 | 跳过生长发育模块 |
| 从未接种过疫苗 | 疫苗进度显示"0/X 待接种"（不跳过模块，提醒用户关注） |
| 从未记录里程碑 | 跳过里程碑模块 |
| AI 评语为空 | 跳过 AI 建议模块 |
| 头像 URL 失效 | 继续使用名字首字母占位符（已有逻辑） |
| Canvas 内存不足 | 捕获异常，提示用户重试，降级为仅生成核心指标卡（不含可选模块） |
| 趋势数据获取失败 | 四维指标卡仅显示数值，不显示状态标签和范围条 |
