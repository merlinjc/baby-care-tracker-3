# Baby Care Tracker 4.0 UI 设计方案

> **版本**: v4.0-r2 | **日期**: 2026-04-10 | **设计师**: UI Design System
> **设计原则**: 少即是多 -- 一次点击完成记录，一眼看清宝宝状态
>
> **配套文档**: [detailed-spec.md](./detailed-spec.md) -- 全部 18 个页面 + 15 个组件的逐一细化规格

---

## 0. 设计总纲

### 0.1 核心原则

| 原则 | 说明 |
|------|------|
| **一次点击** | 首页快捷入口直接打开对应记录弹窗，不设中间层 |
| **一眼可读** | 关键数据用大字号 + 功能色，3 秒内获取信息 |
| **零学习成本** | 保留用户已有操作习惯，渐进式优化视觉层级 |
| **暗色友好** | 暖夜模式下每个元素都经过对比度验证 |

### 0.2 目标用户场景

| 场景 | 频率 | 痛点 | 4.0 解法 |
|------|------|------|---------|
| 凌晨喂奶后记录 | 3-8 次/天 | 步骤多、亮度刺眼 | 首页一键打开弹窗 + 暖夜模式 |
| 查看今天喂了几次 | 5+ 次/天 | 需要数数/计算 | 摘要卡片大字号直觉展示 |
| 睡眠开始/结束 | 2-4 次/天 | 需要开弹窗操作 | 状态胶囊一键开始/结束 + 自动记录 |
| 疫苗到期提醒 | 2-3 次/月 | 容易遗忘 | 首页待办条 + 红点强提醒 |

### 0.3 与现有架构的兼容性

| 维度 | 现有方案 | 4.0 方案 | 变更程度 |
|------|---------|---------|---------|
| 框架 | 微信小程序原生 | 不变 | 无 |
| 样式体系 | CSS 变量 + WXSS | 扩展变量，不改底层 | 低 |
| 组件库 | 15 个自定义组件 | 新增 1 个，重构 2 个 | 低 |
| 服务层 | 11 个单例服务 | 不变 | 无 |
| 图标体系 | PNG 图标 + icon-config.js | 不变，通过 Iconify 补充缺失 | 低 |
| 暗色模式 | ThemeManager + CSS 类 | 不变 | 无 |

### 2.1 布局对比

```

│  问候语           │              │  A. 问候区 + 日期       │
│  宝宝卡片         │              │  B. 状态胶囊(睡眠/喂养) │
```

### 2.3 区域 B: 状态胶囊（核心交互升级）

**v3.x 的状态横幅只是展示信息。v4.0 的状态胶囊是可操作的控件，直接承载睡眠开始/结束并同步写入记录。**

#### 三种状态

```
无睡眠时（默认）:
┌──────────────────────────────────────────────┐
│  [sleep-color.png] 上次喂养 1h32m前          │
└──────────────────────────────────────────────┘

睡眠中:
┌──────────────────────────────────────────────┐
│  [sleep-color.png] 正在睡觉 · 2h30m  [结束]  │
│  ● REC                                       │
└──────────────────────────────────────────────┘

睡眠异常(>24h):
┌──────────────────────────────────────────────┐
│  [warning.png] 睡眠时间异常        [取消计时]  │
└──────────────────────────────────────────────┘
```

#### 交互流程

**开始睡眠**（通过快捷入口触发，非胶囊）:
1. 用户点击快捷入口中的「睡眠」按钮
2. 如果当前无活动睡眠 → 弹出睡眠弹窗，用户选择类型后开始计时
3. 状态胶囊自动切换为「睡眠中」状态
4. 胶囊显示实时时长 + 录制指示灯动画

**结束睡眠**（通过胶囊触发）:
1. 用户点击胶囊右侧「结束」按钮
2. 调用 RecordService.createRecord() 写入睡眠记录（含 startTime/endTime/duration）
3. 更新本地缓存 + 今日统计
4. 胶囊动画过渡回默认状态
5. 时间线自动刷新显示新记录

**取消异常睡眠**:
1. 点击「取消计时」→ wx.showModal 二次确认
2. 清除 active_sleep 缓存，不写入记录
3. 胶囊回到默认状态

#### 规格

| 元素 | 规格 |
|------|------|
| 胶囊容器 | 高度 72rpx / `--radius-md`(24rpx) / margin-bottom: 20rpx |
| 默认背景 | `rgba(212, 184, 150, 0.08)` |
| 睡眠中背景 | `rgba(184, 168, 212, 0.12)` / 睡眠功能色 |
| 异常背景 | `rgba(232, 84, 84, 0.10)` |
| 左侧图标 | 36rpx / 使用 `sleep-color.png` 或 `warning.png` |
| 状态文字 | 28rpx / 500 / `--text-primary` |
| 录制指示灯 | 12rpx 红色圆点 + 呼吸动画 / 左下角 |
| 「结束」按钮 | 胶囊形 / `--primary-color` 背景 / 白字 24rpx / 高度 52rpx |
| 「取消计时」按钮 | 胶囊形 / `--danger-color` 背景 / 白字 24rpx |

#### 暗色模式

| 元素 | 亮色 | 暗色 |
|------|------|------|
| 默认背景 | `rgba(212, 184, 150, 0.08)` | `rgba(212, 184, 150, 0.10)` |
| 睡眠中背景 | `rgba(184, 168, 212, 0.12)` | `rgba(148, 136, 180, 0.15)` |
| 异常背景 | `rgba(232, 84, 84, 0.10)` | `rgba(212, 72, 72, 0.12)` |

```
│  │120ml│  │达标  │  │湿3脏1│  │正常  │        │
```

**相比 v3.x 的变更**:

| 维度 | v3.x | v4.0 | 说明 |
|------|------|------|------|
| 主数字 | 40rpx/700 | **48rpx/700** | 数据更突出 |
| 图标区 | 72rpx 方形 | 64rpx 圆形 | 更轻巧 |
| 辅助行 | 2 行（详情 + 时间距） | **1 行**（仅关键详情） | 减少噪音 |

| 点击 | 跳转筛选 | **直接打开对应弹窗** | 更快速记录 |

| 元素 | 规格 |
|------|------|
| 卡片 | `--bg-secondary` / `--radius-lg` / `--shadow-card` / padding: 28rpx |

| 图标 | 36rpx / 使用 `feeding-color.png` 等已有彩色图标 |
| 主数字 | **48rpx / 700** / `--text-primary` |
| 单位文字 | 22rpx / 400 / `--text-secondary` |
| 辅助行 | 22rpx / 500 / 对应功能色 |

**进度条逻辑**:
- 喂养: 当日次数 / 8 (参考值)
- 睡眠: 当日时长 / NSF 标准
- 排便: 当日次数 / 6 (参考值)
- 体温: 无进度条，改为横线

**点击交互**: 点击任意一列 → 直接打开对应记录弹窗（喂养/睡眠/排便/体温）。
