# 微信小程序发布操作手册

> **版本**: v1.0 | **更新日期**: 2026-04-14 | **状态**: 生效中

---

## 概述

本文档定义微信小程序从代码完成到正式上线的完整发布流程。与 `git-flow.md` 的 Release 分支流程配合使用。

---

## 1. 发布前准备

### 1.1 代码侧检查（Git Flow Release 分支）

参照 `git-flow.md` § 3 Release 发版流程，确保：

- [ ] Release 分支已创建（`release/vX.Y.Z`）
- [ ] `CHANGELOG.md` 已更新
- [ ] 版本号同步清单已全部更新（参照 `development-workflow.md` § 3.3.4）
- [ ] 所有 Spec 的 `tasks.md` 状态为 ✅ 已完成

### 1.2 功能验证

- [ ] 微信开发者工具编译通过（无报错、无严重警告）
- [ ] 核心功能手动测试通过（喂养/睡眠/排便/体温/生长 五类记录）
- [ ] 暗色模式兼容测试（light / dark / 跟随系统）
- [ ] 多宝切换场景测试
- [ ] 家庭协作场景测试（邀请/加入/权限）
- [ ] 离线模式基本可用（断网记录 → 恢复后同步）
- [ ] 分包页面正常加载（packageGrowth / packageSocial）

---

## 2. 上传代码

### 2.1 微信开发者工具上传

```
1. 打开微信开发者工具
2. 确认项目编译无错误
3. 点击「上传」按钮（工具栏右侧）
4. 填写版本信息：
   - 版本号：X.Y.Z（与 CHANGELOG 一致，不带 v 前缀）
   - 项目备注：<版本代号> — <一句话描述>
     示例：Milo — 美拉德 UI 重设计 + 暗色模式
5. 确认上传
```

### 2.2 上传后验证

- [ ] 微信公众平台 → 管理 → 版本管理 中能看到刚上传的版本
- [ ] 版本号和备注正确

---

## 3. 体验版测试

### 3.1 设置体验版

```
1. 登录微信公众平台 (mp.weixin.qq.com)
2. 管理 → 版本管理 → 开发版本
3. 找到刚上传的版本，点击「选为体验版」
4. 使用体验版二维码进行测试
```

### 3.2 体验版测试清单

| 测试维度 | 测试项 | 通过 |
|----------|--------|------|
| **基础功能** | 五类记录（喂养/睡眠/排便/体温/生长）创建、编辑、删除 | |
| **首页** | 问候语、今日概览、快捷记录、AI 洞察、时间线 | |
| **记录页** | 筛选、搜索、批量操作、分享报告 | |
| **发现页** | 疫苗追踪、生长曲线、里程碑、AI 助手 | |
| **我的页** | 用户资料编辑、宝宝管理、家庭协作、数据导出、设置 | |
| **暗色模式** | 三种模式切换、所有页面视觉正常 | |
| **离线** | 断网记录 → 联网同步 | |
| **边界** | 新用户引导、无数据空状态、多宝切换 | |
| **性能** | 首页加载流畅、TabBar 切换无卡顿 | |

## 4. 提交审核

### 4.1 提交步骤

```
1. 微信公众平台 → 管理 → 版本管理 → 开发版本
2. 点击「提交审核」
3. 填写审核信息：
   - 版本说明：列出本次主要更新内容（2-3 条）
   - 功能页面：选择需要审核的核心页面路径
   - 隐私声明：确认符合隐私协议
4. 提交
```

### 4.2 审核注意事项

- 审核周期通常 1-7 个工作日
- 确保所有功能页面可正常访问（无空白页、无报错弹窗）
- 如有获取用户信息/位置等敏感权限，确保有合理的使用场景说明
- 小程序描述和截图需与实际功能一致

---

## 5. 正式发布

### 5.1 审核通过后发布

```
1. 微信公众平台 → 管理 → 版本管理 → 审核版本
2. 确认审核通过
3. 点击「发布」（全量发布）
   或选择「灰度发布」（按比例逐步放量）
```

### 5.2 灰度发布策略（推荐）

| 阶段 | 放量比例 | 持续时间 | 观察指标 |
|------|----------|----------|----------|
| 第一批 | 10% | 1-2 天 | 错误率、崩溃率、关键功能使用率 |
| 第二批 | 50% | 1-2 天 | 用户反馈、性能指标 |
| 全量 | 100% | — | 确认无异常后全量推送 |

### 5.3 发布后验证

- [ ] 正式版可正常打开和使用
- [ ] 版本号显示正确（我的页底部）
- [ ] 新功能正常可用
- [ ] 无明显性能退化

---

## 6. 发布后收尾

### 6.1 Git 操作

参照 `git-flow.md` § 3.3 完成：

```bash
# 合并 release 到 master + 打标签 + 合并回 develop + 推送 + 清理分支
```

### 6.2 通知

- 在团队群/文档中通知新版本已上线
- 更新版本历史和 CHANGELOG

---

## 7. 紧急回滚

如发布后发现严重问题：

```
1. 微信公众平台 → 管理 → 版本管理
2. 找到上一个正式版，点击「版本回退」
3. 确认回退后，立即走 Hotfix 流程修复问题
```

> **注意**：版本回退每天只能操作 1 次，回退后需重新提交审核才能再次发布。
