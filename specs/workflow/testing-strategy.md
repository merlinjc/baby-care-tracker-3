# 测试策略与规范

> **版本**: v1.0 | **更新日期**: 2026-04-14 | **状态**: 生效中

---

## 概述

Baby Care Tracker 项目以**手动测试**为主（微信小程序原生开发暂无自动化测试框架集成），通过标准化的测试清单和回归测试范围确保每次迭代质量。

---

## 1. 测试层级

| 层级 | 方式 | 时机 | 责任 |
|------|------|------|------|
| **功能测试** | 手动验证 | 每个 Task 完成后 | 开发者 |
| **集成测试** | 手动验证 | 每个 Milestone 完成后 | 开发者 |
| **回归测试** | 按清单验证 | Release 分支上 | 开发者 |
| **体验版测试** | 真机验证 | 提交审核前 | 开发者 + 测试人员 |

---

## 2. Feature 测试清单模板

每个 Spec 的 `tasks.md` 中每个 Task 包含验收标准。完成 Task 后按以下模板验证：

```markdown
### T-x.x 验收

- [ ] 功能行为符合 FR-x 验收标准
- [ ] 亮色模式下视觉正常
- [ ] 暗色模式下视觉正常
- [ ] 边界条件处理正确（空数据、超长文本、网络异常）
- [ ] 无 console.error 输出
- [ ] 编译无新增警告
```

## 3. 核心功能回归测试清单

每次 Release 前必须执行以下回归测试：

### 3.1 记录功能（5 类）

| 记录类型 | 测试项 |
|----------|--------|
| **喂养** | 母乳（左/右/双侧）、配方奶（快捷累加 + 清零）、辅食（手动输入）、时长累加 |
| **睡眠** | 类型选择（夜间/午睡）、时长记录、计时器功能 |
| **排便** | 类型选择（尿湿/脏/混合）、颜色/性状选择 |
| **体温** | 测量方式选择、数值输入（35.0~42.0）、异常高温提示 |
| **生长** | 身高/体重/头围输入、WHO 曲线对照 |

### 3.2 页面功能

| 页面 | 关键测试项 |
|------|-----------|
| **首页** | 问候语动态变化、今日概览数据准确、快捷记录入口、AI 洞察加载、时间线展示 |
| **记录页** | 筛选切换、搜索、批量管理、FAB 菜单添加记录 |
| **发现页** | 疫苗追踪、生长曲线 Canvas 绘制、里程碑追踪、AI 助手对话 |
| **我的页** | 个人资料编辑、宝宝管理、家庭协作入口、数据导出、设置、版本号显示 |

### 3.3 跨功能测试

| 维度 | 测试项 |
|------|--------|
| **多宝切换** | 切换宝宝后所有数据刷新正确、缓存不串扰 |
| **暗色模式** | 三种模式切换、所有页面/弹窗/组件视觉正常 |
| **离线模式** | 断网创建记录 → 联网后自动同步 → 数据一致 |
| **权限** | viewer 不能编辑、editor 可编辑不可管理、admin 全权限 |
| **分包加载** | packageGrowth / packageSocial 页面正常加载 |

## 5. 性能基线

| 指标 | 基线值 | 测量方式 |
|------|--------|----------|
| 首页首次加载 | < 2s | 微信开发者工具「体验评分」 |
| TabBar 页面切换 | < 500ms | 主观感知无卡顿 |
| 记录列表滚动 | 60fps | 微信开发者工具「性能监控」 |
| 弹窗打开动画 | 流畅 | 主观感知无跳帧 |
| 分享图生成 | < 3s | 点击到图片显示的时间 |
