# 微信公众号代码块使用说明

## 一、已完成的优化

### 1. 代码块样式优化
- **修改前**: 深色背景 + 特殊字体,微信公众号不支持
- **修改后**: 浅色背景 + 系统字体,兼容性更好

### 2. 代码块内容优化
- **文件结构展示**: 改用 tip 提示框 + 白色背景区块
- **指令示例**: 改用表格形式,左右对比更清晰
- **行内代码**: 使用浅黄色背景 + 边框,视觉更友好

## 二、微信公众号编辑器使用指南

### 方法一:直接复制粘贴(推荐)
1. 在浏览器中打开 `ai-coding-methodology.html`
2. 全选并复制所有内容(Ctrl/Cmd + A, Ctrl/Cmd + C)
3. 在公众号编辑器中直接粘贴(Ctrl/Cmd + V)
4. 检查代码块显示效果,必要时微调

### 方法二:使用公众号编辑器代码块功能
1. 在公众号编辑器中,点击"代码块"图标
2. 将代码内容粘贴进去
3. 选择合适的代码语言(可选)
4. 编辑器会自动格式化

**注意**: 公众号的代码块功能比较简单,建议使用方法一

## 三、代码块样式对比

### 原始样式(不推荐)
```
深色背景 #1e1e2e
白色文字 #cdd6f4
等宽字体 SF Mono
```
**问题**: 公众号可能不支持自定义字体和深色背景

### 优化后样式(推荐)
```
浅灰背景 #f8f8f8
深色文字 #333
系统等宽字体
左边框强调
```
**优势**: 
- ✅ 兼容性好,所有浏览器都支持
- ✅ 移动端显示效果佳
- ✅ 可复制文字内容
- ✅ 视觉层次清晰

## 四、进一步优化建议

### 1. 代码截图方案(效果最好)
如果你的代码块在公众号中仍然显示不理想,可以:
1. 在编辑器中打开代码
2. 使用 Carbon (https://carbon.now.sh) 创建漂亮的代码截图
3. 将截图插入公众号文章中

### 2. 表格方案(结构化内容)
对于结构化的代码说明,可以继续使用表格:
```html
<table>
  <tr>
    <th>指令</th>
    <th>说明</th>
  </tr>
  <tr>
    <td>开始 Day X...</td>
    <td>AI 会读取 design.md...</td>
  </tr>
</table>
```

### 3. 引用块方案(简单代码)
对于简短的代码片段,可以使用引用块:
```html
<blockquote>
  开始 Day X,确保实施和设计方案对齐
</blockquote>
```

## 五、常见问题

### Q1: 复制到公众号后代码块样式丢失怎么办?
**A**: 公众号会过滤部分自定义样式。建议:
- 使用内联样式(已优化)
- 或者使用公众号自带的代码块功能
- 或者使用代码截图

### Q2: 移动端代码块显示不全怎么办?
**A**: 
- 已优化为 `white-space: pre-wrap` 自动换行
- 已设置 `word-wrap: break-word` 防止溢出
- 必要时可以横向滚动查看

### Q3: 代码中的特殊字符显示异常?
**A**: 
- 已使用 HTML 实体编码(如 `&lt;` 代替 `<`)
- 复制时注意保持编码格式
